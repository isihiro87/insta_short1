# 問題チェック

qas.mdファイルの問題を以下の観点でチェックし、**よりわかりやすい問題文を積極的に提案**します。

**レビュー方針**: 「問題がないからOK」ではなく、「もっとわかりやすくできないか？」という視点で全問題を見直す。

## 引数

- `$ARGUMENTS`: quizフォルダパス（例: `datas/history/grade2/4-1/3nanban_trade/quiz`）

---

## ステップ1: ファイルの読み込み

### 1.1 qas.mdファイルの検索

```bash
ls [フォルダパス]/*qas*.md
```

### 1.2 全ファイルの読み込み

各ファイルを読み込み、問題・答え・補足を抽出。

**qas.mdの構造**:
```
問題1
答え1
補足1

問題2
答え2
補足2
```

---

## ステップ2: 問題の改善チェック

各問題を以下の観点で**積極的に改善案を検討**:

### チェック項目

1. **答えが一意に定まるか**
   - NG: 「織田信長の政策は？」→ 複数回答が可能
   - OK: 「織田信長が楽市楽座を行った城下町は？」→「安土」と一意に定まる

2. **中学生の学習範囲に適切か**
   - 教科書に載っている用語・概念か
   - 高校範囲や専門的すぎる内容ではないか

3. **文章表現が適切か**
   - 難しい熟語や専門用語を使いすぎていないか
   - 一文が長すぎないか（目安: 35文字以内）
   - 主語・述語が明確か

4. **問題文の形式が自然か**
   - 「〜は？」形式に縛られず、自然な日本語で問う
   - **答えの形式を問題文に含める**ことで、何を答えればいいか明確にする
   - **曖昧な問題には補足説明を追加**: 複数の解釈が可能な場合は `(改名後を答える)` のような注釈を入れる

   **改善パターン例**:
   | 改善前 | 改善後 |
   |--------|--------|
   | 1603年に徳川家康が江戸に開いた武家政権は？ | 1603年に徳川家康が開いたのは何幕府？ |
   | 1600年に家康が石田三成らを破った戦いは？ | 1600年に家康が石田三成らを破ったのは何の戦い？ |
   | 徳川家の一族である大名の区分は？ | 徳川家の一族である大名を何という？ |
   | 幕府が大名を統制するために定めた法律は？ | 大名を統制するために幕府が定めた法律を何という？ |
   | 朝鮮との国交回復に尽力した藩は？ | 朝鮮との外交窓口となったのは何藩？ |
   | 1637年に九州で起こった大規模な反乱は？ | 1637年に九州で起きたのは何という一揆？ |
   | ロシア革命後に改名した党名は？ | ロシア革命後に改名した党の名前は？(改名後を答える) |

   **使える表現パターン**:
   - 「何〇〇？」: 答えが「△△〇〇」の形式の場合（何幕府？→江戸幕府）
   - 「〜を何という？」: 用語を問う場合
   - 「〜は何？」: 具体的なものを問う場合
   - 「どこ？」「だれ？」: 場所・人物を問う場合

   **避けるべき表現**:
   - 「武家政権」「区分」など中学生に難しい言葉
   - 回りくどい表現（「〜させたのは？」など）

5. **改行ルールの確認**（レンダリング時に重要）
   - 問題文に `\n` を入れる場合、**各行は11文字以内**（12文字以上は禁止）
   - 文字数の数え方:
     - 日本語（ひらがな、カタカナ、漢字、句読点、括弧）: 1文字 = 1
     - 半角英数字: 2文字 = 1（例: 1603 = 2文字分）
   - 11文字を超える場合は、別の位置で区切るか、改行なしで自動改行に任せる

   **改行位置の判断基準**:
   - **9〜11文字**で切りの良い位置がある → `\n`を入れる（推奨）
   - **8文字以下**でしか切れない → `\n`を入れない（自動改行に任せる）
   - **12文字以上**でしか切れない → `\n`を入れない（自動改行に任せる）

   **改行の例**:
   | 問題文 | 改行位置 | 文字数 |
   |--------|---------|-------|
   | ロシア革命を指導した社会主義者は？ | ロシア革命を指導した\n社会主義者は？ | 10文字 + 8文字 ✅ |
   | レーニンの死後、ソ連の最高指導者となったのは？ | レーニンの死後、ソ連の\n最高指導者となったのは？ | 11文字 + 11文字 ✅ |
   | ロシア革命後に改名した党の名前は？(改名後を答える) | ロシア革命後に改名\nした党の名前は？\n(改名後を答える) | 9文字 + 8文字 + 9文字 ✅ |

6. **補足説明のスタイル**
   - **言い切り形式を使う**（敬語「〜です。」「〜ます。」は使わない）
   - 簡潔で読みやすい表現にする

   **修正パターン例**:
   | 修正前（敬語） | 修正後（言い切り） |
   |----------------|-------------------|
   | 低温で焼かれたため黒褐色で厚手のものが多いです。 | 低温で焼かれたため黒褐色で厚手のものが多い。 |
   | マンモスなどの大型動物を追って人々が移動していました。 | マンモスなどの大型動物を追って人々が移動していた。 |
   | 当時の人々の食べ物や生活を知る貴重な資料となります。 | 当時の人々の食べ物や生活を知る貴重な資料となる。 |

---

## ステップ3: 改善案の実施

チェック結果に基づき、**全問題に対して改善を検討**:

- **改善案がある問題**: より分かりやすい表現に修正
- **すでに最適な問題**: そのまま保持（ただし「最適である理由」を意識する）

**積極的に改善すべきポイント**:
- 答えの形式が問題文から推測しにくい → 「何〇〇？」形式に変更
- 難しい言葉がある → 中学生にわかりやすい言葉に置き換え
- 長い問題文 → 簡潔に言い換え
- 曖昧な問いかけ → 補足説明 `(〜を答える)` を追加
- 補足が敬語になっている → 言い切り形式に変更

---

## ステップ4: ファイル出力

### 4.1 修正済みqas.mdの生成

修正を適用したqas.mdファイルを生成。

**保存先**: `[フォルダパス]/qas_reviewed.md`

**フォーマット**:
```
修正後の問題1
答え1
補足1

修正後の問題2
答え2
補足2
```

※ 改善の余地がない問題は元のまま含める。

### 4.2 レビュー記録の生成

全問題の検討結果と修正内容を記録。

**保存先**: `[フォルダパス]/qas_review.md`

**フォーマット**:
```markdown
# 問題レビュー記録

生成日: [日付]
対象: [元ファイル名]

## サマリー

- 全問題数: [N]問
- 修正なし: [X]問
- 修正あり: [Y]問

---

## 検討内容

### Q[番号]: [修正あり/現状維持]

**元の問題文**:
[元の問題文]

**改善後**:（修正ありの場合）
[修正後の問題文]

**検討内容**:
[改善した理由、または現状が最適と判断した理由]

---

### Q[番号]: [修正あり/現状維持]
...
```

※ 修正がなかった問題も「検討したが現状が最適」と判断した理由を簡潔に記載。
