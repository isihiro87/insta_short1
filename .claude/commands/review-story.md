# 脚本添削

既存の脚本（kyakuhon.txt）をルールに基づいて添削し、改善案を提示します。

## 引数

- `$ARGUMENTS`: storyフォルダパス（例: `datas/history/grade2/4-1/4nobunaga_hideyoshi/story/6_honnoji`）

---

## フェーズ1: 脚本の読み込み

### 1.1 ファイルの確認と読み込み

```
[フォルダパス]/kyakuhon.txt
```

脚本を読み込み、以下を抽出:
- N1/N2/N3のナレーション行
- 説明行
- 全体の行数

---

## フェーズ2: ルールに基づくチェック

**重要: 問題がなくても、より良い表現があれば積極的に提案する**

### 2.1 ナレーション行のチェック

| チェック項目 | 基準 |
|------------|------|
| N1の形式 | 「次が気になる文」になっているか（「○○は」「○○があり」「○○だったころ」） |
| 文字数 | 25字以内か |
| N2/N3の役割 | N2は転換点、N3は結論・余韻になっているか |

**N1の推奨パターン**:
- `〜だったころ` → 「何があったの？」と思わせる
- `〜には○○があり、` → 「それで？」と続きを期待させる
- `〜に対して○○は` → 「何をしたの？」と興味を引く
- `○○は` → 次の行への期待を作る

**N1のNGパターン**:
- 完結形（「〜した」「〜だった」で終わる）→ 続きへの期待が薄れる

### 2.2 説明行のチェック

| チェック項目 | 基準 |
|------------|------|
| 文字数 | 30字以内を目安（最大40字） |
| 1行1情報 | 1行に複数の情報を詰め込んでいないか |
| 一文の長さ | 読点「、」が2つ以上あれば分割を検討 |

### 2.3 わかりやすさのチェック（重要）

**すべての行について以下を検討し、より良い表現を積極的に提案:**

| チェック項目 | 基準 | 改善例 |
|------------|------|--------|
| 難しい言葉 | 中学生が一発で理解できるか | 「奇襲」→「不意打ち」「だまし討ち」 |
| 長い文 | 一文が長すぎないか（20字超えたら要検討） | 文を分割する |
| 主語の省略 | 誰の話か明確か | 主語を補う |
| 抽象的な表現 | 具体的にイメージできるか | 数字や比喩を使う |
| 受け身表現 | 能動態にできないか | 「〜された」→「〜した」 |
| 回りくどい表現 | もっとシンプルに言えないか | 不要な修飾語を削る |

**長い文の分割例**:
- NG: `信長は奇襲攻撃をしかけ、今川義元に逆転勝利した`（23字）
- OK: `信長は不意打ちをしかけた` + `そして今川義元に逆転勝利`

**わかりやすい表現の例**:
- `全国統一を目前に` → `天下統一まであと少し`
- `裏切りを起こした` → `裏切った`
- `逃げ場を失い` → `逃げられなくなり`

### 2.4 全体構成のチェック

| チェック項目 | 基準 |
|------------|------|
| 行数 | 合計8-10行程度か |
| 流れ | N1（導入）→ N2（展開）→ N3（結論）の流れが自然か |
| 中学生向け | 難しい用語に補足があるか |

### 2.5 史実チェック

| チェック項目 | 確認内容 |
|------------|---------|
| 年号・時期 | 出来事の年代が正しいか |
| 人物 | 登場人物の役割・行動が正確か |
| 因果関係 | 出来事の原因と結果が史実と一致しているか |

## フェーズ4: 修正案ファイルの作成

### 4.1 修正案ファイルの生成

添削結果と改善案をまとめた `kyakuhon_review.md` を同じフォルダに作成:

```
[フォルダパス]/kyakuhon_review.md
```

ファイルの内容:
```markdown
# 脚本添削レポート

## 元の脚本
[kyakuhon.txtの全文]

## 添削結果

### ナレーション行
| 行 | 現在 | 文字数 | 判定 | コメント |
|----|------|--------|------|---------|
| N1 | ... | ...字 | ○/△/× | ... |
| N2 | ... | ...字 | ○/△/× | ... |
| N3 | ... | ...字 | ○/△/× | ... |

### 説明行
[チェック結果]

### わかりやすさ
[各行について、より良い表現の提案]

### 全体構成
[チェック結果]

### 史実
[チェック結果]

## 改善案

**すべての行について、より良い表現を提案:**

### [行番号]
**現在:** ...
**改善案:** ...
**理由:** ...

## 修正後の脚本案

[改善案を反映した脚本の全文]
```

### 4.2 フォルダの作成

同じフォルダ内に空の `audio/` と `images/` フォルダを作成:

```bash
mkdir -p [フォルダパス]/audio
mkdir -p [フォルダパス]/images
```

## 参考: N1/N2/N3の例

### 例1: 楽市・楽座

```
N1: 信長が登場する前、商売には2つの大きな壁があり、
一つ目、特定の商人だけが商売できる「座」という組合があった
二つ目、道の途中で通行料を支払うため商品の値段が高くなっていた
N2: 自由に商売ができず、経済が発展しにくい状態に対して信長は
安土城の城下町で楽市・楽座を実施
誰でも自由に市場で商売してOKに
「座」の特権を廃止
さらに、領内の関所を廃止
N3: 「自由に商売できる仕組み」を作って経済を発展させ、その力で天下統一を進めた
```

**ポイント**: N1は「〜があり、」で次が気になる形

### 例2: 桶狭間の戦い

```
N1: 織田信長が尾張の小さな戦国大名だったころ
約2万5千の大軍を率いる今川義元に対し
信長の兵は3千〜5千人ほどしかいなかった
N2: 序盤は当然苦戦したが、今川軍の油断と激しい雨が降った隙を逃さず、
信長は奇襲攻撃をしかけ、今川義元に逆転勝利
この戦いをきっかけに今川の支配下にあった徳川家康は独立
のちの同盟につながった
N3: この「桶狭間の戦い」で織田信長は一躍有名になり、天下統一への第一歩を踏み出した
```

**ポイント**: N1は「〜だったころ」で時代設定しつつ次が気になる形

### 例3: 本能寺の変

```
N1: 1582年、全国統一を目前にしていた織田信長は
しかし、信長が本能寺に滞在していたある早朝
家臣の明智光秀が突然裏切りを起こした
N2: 光秀は「敵は本能寺にあり」と言って軍を率い
本能寺を取り囲み、奇襲攻撃をしかけた
信長は逃げ場を失い、本能寺に火を放って自害した
N3: なぜ光秀が裏切ったのかは今も謎で、怨恨説や野望説など様々な説がある
光秀は本能寺の変からわずか11日後に秀吉に敗れ、「三日天下」と呼ばれている
```

**ポイント**: N1は「○○は」で次が気になる形

## 重要なルール

1. **積極的に提案** - 問題がなくても、より良い表現があれば必ず提案する
2. **一文は短く** - 20字を超えたら分割を検討。読点「、」2つ以上はNG
3. **わかりやすく** - 中学生が一発で理解できる表現を選ぶ
4. **N1は次が気になる文に** - 「○○は」「○○があり」のように続きを期待させる
5. **ナレーション行は25字以内** - 説明行は30字以内を目安
6. **史実チェック** - 年号・人物・因果関係が正確か確認
7. **改善案は具体的に** - 何をどう直すか、なぜ直すか明確に提示
