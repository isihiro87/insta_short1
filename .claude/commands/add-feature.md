# 新機能の追加

**引数:** 機能名 (例: `/add-feature 新しいクイズテーマ`)

---

## ステップ1: 準備とコンテキスト設定

1. 現在のタスクコンテキストを確立する:
   - 機能名: `$ARGUMENTS`
   - 日付: `[現在の日付をYYYYMMDD形式で取得]`
   - ステアリングディレクトリパス: `.steering/[日付]-[機能名]/`

2. ステアリングディレクトリを作成し、以下の3ファイルを作成:
   - `requirements.md`
   - `design.md`
   - `tasklist.md`

## ステップ2: プロジェクト理解

1. `CLAUDE.md`を読み、プロジェクトの全体像を把握する
2. `docs/`ディレクトリ内の永続ドキュメントを確認する

## ステップ3: 既存パターンの調査

1. Grepツールで関連キーワードを`src/`内で検索
2. 既存の実装パターン、命名規則を特定

## ステップ4: 計画フェーズ

1. `Skill('steering')`を計画モードで実行
2. requirements.md, design.md, tasklist.mdの内容を生成

## ステップ5: 実装ループ

**tasklist.mdの全タスクが完了するまで繰り返す:**

1. tasklist.mdを読み込み、未完了タスク（`[ ]`）を特定
2. タスクを実装
3. 完了したタスクを`[x]`に更新
4. 次のタスクへ

## ステップ6: 検証

1. `npm run lint` でLintチェック
2. `npm run dev` で動作確認

## ステップ7: 振り返り

1. `Skill('steering')`を振り返りモードで実行
2. tasklist.mdに振り返りを記録

## 完了条件

- tasklist.mdの全タスクが完了（`[x]`）
- Lintエラーなし
- 振り返りが記録されている
