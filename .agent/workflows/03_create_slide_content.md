2.  **ファイル作成**

      - 各 `video_content[数字].md` に対応する `slide_content[数字].md` を作成する（連番を一致させる）。

# 補足資料

## slide_content.md 作成ルール

### 1. 基本方針：行単位のタイムライン制御

`video_content.md` の読み上げ原稿は、音声の「間」や「テンポ」を作るために細かく改行されています。
スライド資料作成時は、**勝手に行を結合せず、原稿の1行をテーブルの1行として扱い**、その瞬間の画面上の文字情報の状態を定義してください。

### 2. ファイル形式・構造

  - **ファイル名**: `slide_content[数字].md`
  - **フォーマット**: Markdown テーブル形式
  - **参照資料**: レイアウトの詳細は `src/EnglishLessonPatterns.md` を参照すること。

| 行No | レイアウト | 読み上げ原稿 | スライドテキスト | 補足セリフ |
| :--- | :--- | :--- | :--- | :--- |
| 1 | standard | 問題！ | 次の日本語を英語にしなさい。 | さあ、解けるかな？ |
| 2 | standard | カッコに入る単語は... | 次の日本語を英語にしなさい。<br><br>このペンはあのペンより長いです。<br>This pen is ( ) ( ) than that pen. | |
| 3 | horizontal-split | まず、日本語の... | Left: 元の形<br>long<br><br>Right: 比較級<br>longer | ここがポイントだよ！ |

### 3. カラム記述ルール

#### A. レイアウト (Layout)
  - `src/EnglishLessonPatterns.md` の「レイアウト使い分けガイド」に基づき、その行に最適なレイアウトを指定する。
  - **原則として `standard` を使用する**。
  - 比較やリスト表示が必要な場合のみ、他のレイアウト (`horizontal-split`, `vertical-split`, `vertical-list` 等) を選択する。

#### B. スライドテキスト（画面上の文字情報の状態）
  - レイアウトに応じて記述形式を変えること。
  - **standard**: 通常のテキスト記述。改行は `<br>` を使用。
  - **horizontal-split**: `Left: [左の内容]<br><br>Right: [右の内容]`
  - **vertical-split**: `Top: [上の内容]<br><br>Bottom: [下の内容]`
  - **vertical-list**: `・[項目1]<br>・[項目2]`
  - **情報の更新・継承**:
      - `(維持)`: 前の行から変化がない場合。
      - `(維持) + [追記]`: 前の行の内容をベースに追加する場合。

#### C. 補足セリフ（右端に追加）
  - 先生アバターやキャラクターが生徒に語りかけるような「合いの手」「補足」を記述する。
  - 全体の2割程度の行に分散して配置する。

### 4. レイアウト選択の指針

| レイアウト | 推奨される使用シーン |
| :--- | :--- |
| **`standard`** | **基本はこれ**。例文、穴埋め問題、シンプルな解説。 |
| **`horizontal-split`** | **左右対比** (変化前→変化後、肯定→否定)。 |
| **`vertical-split`** | **上下対比** (ルール→例文、NG→OK)。 |
| **`vertical-list`** | **箇条書き** (ポイントまとめ、単語リスト)。 |
| **`quiz-choices`** | **選択問題** (3〜4択)。 |
| **`table`** | **表形式** (活用表など)。 |
| **`dialogue`** | **会話形式** (AさんとBさんの対話)。 |
