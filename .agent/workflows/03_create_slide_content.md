2.  **ファイル作成**

# 補足資料

## slide_content.md 作成ルール

### 1. 基本方針：行単位のタイムライン制御

`video_content.md` の読み上げ原稿は、音声の「間」や「テンポ」を作るために細かく改行されています。
スライド資料作成時は、**勝手に行を結合せず、原稿の1行をテーブルの1行として扱い**、その瞬間の画面上の文字情報の状態を定義してください。

### 2. ファイル形式・構造

  - **ファイル名**: `slide_content[数字].md`
  - **フォーマット**: Markdown テーブル形式
  - **列構成**: 「画像・演出」列は廃止し、**テキスト情報と補足セリフ**に集中する。

| 行No | 読み上げ原稿 | スライドテキスト | 補足セリフ |
| :--- | :--- | :--- | :--- |
| 1 | 問題！ | **【問題】**<br>次の日本語を英語にしなさい。 | さあ、解けるかな？ |
| 2 | カッコに入る単語は... | **【問題】**<br>次の日本語を英語にしなさい。<br><br>このペンはあのペンより長いです。<br>This pen is ( ) than that pen. | |
| 3 | まず、日本語の... | (維持)<br>※「〜より長い」を赤字強調 | ここがポイントだよ！ |

### 3. カラム記述ルール

#### A. 行No と 読み上げ原稿

  - `video_content.md` の「読み上げ原稿」セクションにある**空行以外の全ての行**を、上から順に1行ずつ転記する。
  - **重要**: 複数の行を1つのセルにまとめないこと。

#### B. スライドテキスト（画面上の文字情報の状態）

  - その行が読み上げられている瞬間に、**画面に表示されている「全ての文字情報」**を記述する。
  - **わかりやすさの追求**:
      - 中学生が見てパッと理解できるよう、箇条書きや改行（`<br>`）を積極的に使う。
      - 重要なキーワードは `**太字**` や `[色:赤]` などのテキスト装飾記法を用いて強調箇所を明示する。
  - **情報の更新・継承ルール（重要）**:
      - **`(維持)` の下に追加**: **1つ前（直前）の行**の内容をベースに、要素を追加・変更する場合に使用する。
      - **`(維持-2)` の下に追加**: **2つ前の行**の内容をベースに、要素を追加・変更する場合に使用する。（直前の行で行った一時的な強調やヒント表示を取り消し、元の状態に戻してから次の情報を出す場合などに使う）
      - **`(維持)` のみ**: 前の行から変化がない場合。
  - **代名詞の排除**: 文脈がなくても理解できるよう、`It` や `One` などの代名詞は具体的な単語（例: `pen`, `bag`）に置き換えて記述する。

#### C. 補足セリフ（右端に追加）

  - **役割**: 先生アバターやキャラクターが生徒に語りかけるような「合いの手」「励まし」「補足」を記述する。
  - **頻度**: 全ての行に入れる必要はない。**全体の3割程度**の行に分散して配置すること。
  - **内容例**:
      - 注意喚起：「ここはミスしやすいよ！」
      - 思考の補助：「まずは主語を見つけよう」
      - 承認・励まし：「その調子！」「完璧だね」

### 4. スライド構成の定石（汎用テンプレート）

視覚的な演出指定がない分、**テキストの出し方（順序・強調）**でわかりやすさを担保してください。

#### フェーズ1：問題提示

  - **目的**: 何を問われているかを明確にする。
  - **テキスト構成**:
      - 「見出し（例：練習問題）」
      - 「日本語訳（全文）」
      - 「英文（穴埋め部分は `( )` や `[ ]` で表示）」
  - **工夫**: 英文を読むタイミングで、対応する日本語を太字にするなどの連動を持たせる。

#### フェーズ2：解説プロセス（思考の可視化）

  - **目的**: 正解に至る論理を表示する。
  - **テキスト構成**:
      - 問題文は残したまま、その下に「ヒント」や「公式」を追記していく。
      - `(維持-2)` を活用し、「一時的なヒント表示」→「消去」→「次のヒント」のような切り替えも表現する。
  - **工夫**: 変化する部分（単語の語尾など）を強調表示する記述を含める。

#### フェーズ3：正解とまとめ

  - **目的**: 知識を定着させる。
  - **テキスト構成**:
      - 空欄が埋まった「完全な英文」を表示する。
      - 文法ポイント（S+Vなど）を記号で付記する。

### 5. 記述例（比較級の場合）

| 行No | 読み上げ原稿 | スライドテキスト | 補足セリフ |
| :--- | :--- | :--- | :--- |
| 1 | 問題！ | **【問題】**<br>次の文を英語にしなさい。 | みんな、準備はいい？ |
| 2 | この鉛筆はあの鉛筆よりも長いです。 | **【問題】**<br>この鉛筆はあの鉛筆よりも長いです。<br>This pencil is ( ) than that pencil. | |
| 3 | カッコに入る単語は分かるかな？ | (維持) | 自信を持って答えてね |
| 4 | まず、日本語の「より長い」に注目しよう。 | (維持)<br>↓<br>※「〜よりも長い」を赤字強調 | 比べる言葉があるね |
| 5 | ただの「長い」なら long だけど | (維持-2)<br>↓<br>基本の単語：**long**<br>（※行4の赤字強調を消し、行3の状態に戻してからlongを追加） | |
| 6 | 今回は比べっこしてるよね。 | (維持)<br>↓<br>比較の合図：**er + than** | つまり比較級だ！ |
| 7 | だから正解は longer になります。 | **【正解】**<br>This pencil is **longer** than that pencil.<br>（long に er をつける） | その通り！ |
